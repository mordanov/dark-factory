# Implementation Plan: Frontend Design System

**Branch**: `004-ui-design-system` | **Date**: 2026-05-25 | **Spec**: [spec.md](./spec.md)

## Summary

Refactor the entire React TypeScript frontend (~3,300 LOC, 20 files) from ad-hoc inline CSS to a
production-grade design system using Tailwind CSS v3, Radix UI primitives, and shadcn/ui component
library. Every React `style` prop (141 inline usages + 118 CSSProperties constant objects) is
replaced with Tailwind utility classes. shadcn/ui primitives replace all custom interactive
component implementations (dialogs, dropdowns, inputs, badges, tables). A new `AppShell` layout
component introduces a persistent topbar with a dark mode toggle. The existing `data-theme`/CSS
variable theming system (feature 002) is extended with shadcn/ui HSL variable aliases for full
Tailwind dark mode support. Framer Motion adds page transitions, modal animations, and card hover
feedback. Business logic, API calls, and routing are untouched.

## Implementation Phases

### Phase 1 — Foundation (blocking for all subsequent phases)

Install and configure the toolchain. Nothing else can proceed until these build and render.

**Tasks**:
1. Install Tailwind CSS v3 + PostCSS + Autoprefixer as dev deps; create `tailwind.config.ts` and `postcss.config.js`
2. Replace `index.css` with Tailwind directives (`@tailwind base/components/utilities`); update import order in `main.tsx`
3. Update `themes.css`: convert all hex values to `hsl(HSL-channels)` format; add shadcn/ui CSS variable aliases to every theme block
4. Configure `tailwind.config.ts` with `darkMode: ["class", '[data-theme="dark"]']` and full color token mapping
5. Initialize shadcn/ui: run `npx shadcn@latest init`, create `components.json`, configure `@/` path alias in `vite.config.ts` and `tsconfig.json`
6. Create `src/lib/utils.ts` with `cn()` helper
7. Update `useTheme.ts`: add `.dark` class toggle to `applyTheme()`
8. Install Framer Motion + Lucide React; create `src/lib/motion.ts` with shared animation variants

**Acceptance**: `npm run dev` starts without errors; no console warnings; base `<html>` renders with correct `data-theme` and `dark` class states; `npm run typecheck` passes.

### Phase 4 — Form Component Migration (US3: consistent design system)

Migrate all form controls across 5 pages and relevant components.

**Target files**:
- `src/pages/LoginPage.tsx` — email/password inputs + submit button
- `src/components/tickets/TicketForm.tsx` — title, description, type select, tag input
- `src/components/admin/UserForm.tsx` — email, password, role select
- `src/components/common/FilterBar.tsx` — search input, filter dropdowns

**Pattern**:
```tsx
// Before
<input style={inputStyle} value={val} onChange={...} />

// After
<div className="space-y-2">
  <Label htmlFor="field-id">Label text</Label>
  <Input id="field-id" value={val} onChange={...} />
</div>
```

**Acceptance**:
- All form controls use `<Input>`, `<Textarea>`, `<Label>`, `<Select>` from `ui/`
- Zero `style` prop usage in these files
- Forms remain functionally identical (submit, validation, error states)
